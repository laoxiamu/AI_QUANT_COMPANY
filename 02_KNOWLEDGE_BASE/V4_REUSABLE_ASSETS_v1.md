## A. 与当前 Phase 1 直接相关（可借鉴）

**A1. 方法论铁证：胜率/edge 是信号的结构特征，不是参数能调出来的（FACT，强力佐证）**
V4 回测：39.5%±0.1% 胜率在 1.5%~3.0% 止损范围内几乎不变 →"突破要从信号质量本身入手"。
这与我们 v4/0B17 的结论（止损调参收敛、瓶颈在信号）**独立收敛**。Phase 1 纪律：不在单一信号上反复调参，换信号类别。

**A2. "顺大逆小" HTF 方向约束（可借鉴，与 TSMOM 收敛）**
V4 缺陷5 + D04§3.3：禁止逆更高周期趋势交易（日线跌时禁做多）。这正是 TSMOM 的内核（顺趋势）。
对任何未来信号：加一条日线/4H 方向过滤是低成本高价值的硬过滤。**TSMOM v1 已隐含此原则。**

**A3. 周期匹配（可借鉴，支持 TSMOM 不设紧止损的设计）**
V4 缺陷2："花 20 小时确认趋势，却只给 1.5% 容错"→ 趋势确认周期必须与止损/持仓周期匹配。
TSMOM v1 用 90 天趋势 + 持有至翻转、不设紧止损，正好规避该缺陷。若日后给 TSMOM 加止损，须用宽止损（ATR/结构），不可用窄固定止损。

**A4. 相关性聚簇风险（可借鉴，TSMOM 组合解读用）**
V4 评审5.3 + 我方知识库：BTC/ETH/SOL 高相关 → 可能同时止损/同向回撤，有效独立样本远低于表面笔数。
TSMOM 三品种组合的回撤会聚簇，解读时不可把三品种当独立样本。

**A5. 波动压缩因子的两种用法（参考，注意我方已部分证伪）**
V4 唯一通过 IC 的因子（PC-09）。V4 同时把它当 ①加分项 ②"不做"信号（压缩末端方向爆发易穿止损，D04§3.5）。
注意：我方 0B15 测"压缩作为 Regime 过滤"失败；V4 是"作为打分/不做门槛"，用法不同。若未来重访，按"不做门槛"用，别再当 Regime 过滤。

## C. 明确不借鉴 / 陷阱警告

**C1. ⚠️ V4 的"策略升级方向"几乎全部指向 SMC 结构交易（关键位/流动性扫单/FVG/BOS/CHoCH 四层框架），这正是我们 v1~v5+v4 家族用 8 次实验关闭的死胡同。**
不要因为这些文档写得详尽就重新把 SMC 框架当作前进路线。它在 4H 加密上样本内有边沿、跨期不可部署（DEC-047）。

**C2. ⚠️ 22 分制多因子评分系统 IC≈随机（PC-08），复杂确认链不增预测力。** 不重建评分动物园。

**C3. V4 是现货式价格行为思路，没有永续原生 alpha（资金费率/基差）的概念。** 我们当前的资金费率/TSMOM 方向是 V4 没有的新增量，不在 V4 文档里找它的背书。

**C4. 多周期入场法 / 回踩确认 —— 区分"原则"与"V4 方法"（2026-06-06 评估）**
- **顺大逆小（高周期定方向）= 原则可借，且 TSMOM 已是其纯粹实现**，无需另搬。
- **V4 的多周期确认链入场（1h方向+15m位置+15m确认）= 不借。** ①属已关闭的确认链死路（v1/v2 失败）；②V4 自己的策略审查把它判为"缺陷1 马后炮/缺陷2 周期错配"——是作者自诊为坏的东西。
- **回踩确认 = 现在不借。** ①本质是趋势中的逆势入场，与我们失败的全部信号同家族；②与选 TSMOM 的初衷（纯顺势、与历史失败正交）直接冲突；③V4 里也仅 P3"需IC验证"，从未证实；④关键位/回踩在回测极易前视偏差与过拟合，须机械定义。
- **回踩唯一合理的未来位置**：待 TSMOM v1 证明趋势扣成本后有 edge 后，预登记单变量 v2，在**已确立上升趋势内**用回踩均线作**入场时机优化**（顺势续做、降换手成本），不是独立的回踩反转信号。
- 纪律：不得把任何入场过滤叠加到 TSMOM v1（违反单变量）。

## B'. 系统/架构方案（两份大部头：V4.6.1-Production、V4.6.2-Optimized-Update1）—— Phase 2 蓝图

这两份是 V4 完整的生产级系统设计，价值集中在 Phase 2（建系统）。**性质仍为 HYPOTHESIS 设计蓝图，不是已验证有效的东西。**

**B'1. ★相关性惩罚的 Risk Parity 仓位（直接对应我们反复踩的相关性坑）**
- 基础权重 = 1/ATR%（波动率倒数，SOL 这种高波动自然拿小仓）。
- 相关性惩罚：任两币 72根1h 滚动相关 > 0.8 归一组，**组总风险预算 × 0.6**，组内按原权重二次分配。
- 这是处理 BTC/ETH/SOL 同涨同跌、防止风险预算被重复计算的成熟做法。TSMOM/任何多品种组合到仓位层时直接可借鉴（替代当前等权 1/3）。

**B'2. ★连续函数动态仓位（0B18"按状态缩仓"的成熟版）**
- 回撤指数衰减：`scale = exp(-k×drawdown)`，k≈5（平滑降仓，无悬崖效应）。
- 滚动夏普修正（近20笔）：Sharpe>1.5→×1.2，<0.5→×0.7。
- 波动率熔断：5m ATR/price ≥4% → ×0.5。
- 最终裁剪到 [0.1, 1.5]。
- 0B18 用"波动率百分位÷2"是这套的粗糙残影；若未来重做仓位调节，按这套连续函数设计，且恐惧度代理要对齐"方向失效"而非单纯波动率（DEC-047 教训）。

**B'3. 硬熔断与破产保护**：单日亏损3%、连亏3笔暂停/5笔停盘、回撤20%市价清仓需人工恢复、每月 Monte Carlo 破产概率>5%则下月降险至1.5%。Total risk cap = min(2%, 1−MC_DD_P95)。Kelly×环境系数（趋势0.5/震荡0.25/消息0.15/极端0.0）。

**B'4. 分层架构 + 单向控制铁律（防呆）**：策略→Final Risk Gate 裁决→执行；执行层不得改 symbol/direction/quantity/stop，仅可微调限价 ±0.05%。signed handoff。对 V4 幽灵持仓那类 bug 是结构性防范。

**B'5. 执行层**：TWAP-lite 分批挂单、OBI 动态撤单、滑点>0.1%自动作废并记录。

**B'6. 冷启动四阶段**（直接映射我们 Phase 2→3）：基础设施验证 → 纯规则 Paper(2周) → 影子模式(1周) → 小资金(10%仓位上限)。

**B'7. 策略铁律 11条**（D8.5，可作 Phase 2 纪律清单）：总险≤2%/单笔≤1%/微观结构不独立产生信号/回撤20%清仓需人工/月度MC破产>5%降险/滑点>0.1%作废/子策略连亏10笔禁用/**任何需持久化的规则必须写进配置或Cron，禁止依赖Agent口头承诺**（最后一条对我们 Claude/Codex 协作尤其重要）。

**B'8. ⚠️ Hermes RL（Contextual Bandit / Thompson Sampling）权重自适应、技能自沉淀**：这是学习层，复杂度高、过拟合风险大。**V4 把这套全建了，仍在策略层失败**——因为底层信号无 edge。强烈建议：不在有验证过的信号之前碰 RL 自适应层（属过度工程，触发风险B）。

## C'. ⚠️ 全局最重要的警告（来自这两份方案的存在本身）

**V4 把上面所有精密的仓位/风控/执行/RL/监控/部署机器全建好了，系统也上线运行了，最后仍然失败——因为底层信号（22分制评分）IC≈随机，没有 edge。**
这正是 DEC-004/013「先验证 Alpha 再建系统」和风险B（疯狂建设、忘记验证 Alpha）的活教材。
→ 这些系统蓝图是**有验证过的信号之后**才动用的 Phase 2 资产；**现在（Phase 1）碰它们就是重蹈 V4 覆辙**。当前唯一该做的是先用 TSMOM 找到一个真正扣成本后 Sharpe>1 的信号。

## D. 一句话结论

V4.6.2 对我们最有价值的不是"它的策略"（策略是已关闭的 SMC 死路），而是：**它的纪律与教训**——胜率是结构特征、顺大逆小、周期匹配、相关性聚簇、"不做"条件、Setup 规格化。这些是过程资产，可借鉴；它的 SMC 信号体系不可再投入。
