# v4\.6\.2版本方案部署计划

## V4\.6\.2 系统落地执行计划（新加坡腾讯云单机版）



### 📋 总体规划概览



|阶段|名称|预计耗时|核心目标|
|---|---|---|---|
|**阶段一**|环境准备与安全检查|1\-2天|确保服务器环境干净可用|
|**阶段二**|安装 Hermes Agent 并完成基础配置|2\-3天|让 Hermes 成为您的“部署顾问”|
|**阶段三**|利用 Hermes 引导 OpenClaw 优化|2\-3天|配置 OpenClaw 的记忆与多 Agent 路由|
|**阶段四**|搭建 State Center 与核心基础设施|2\-3天|部署 Redis、PostgreSQL、Grafana|
|**阶段五**|部署 Decision Gateway 与 Execution Engine|3\-5天|构建风控核心与执行层|
|**阶段六**|配置监控、告警与定时任务|2\-3天|让系统可观测、可运维|
|**阶段七**|分步测试与上线|4\-6周|从 Paper Trading 到小额实盘|

### 📅 阶段一：环境准备与安全检查



**目标**：确保服务器环境干净、安全、可追溯，为后续部署打好基础。



|项目|内容|
|---|---|
|**前置条件**|已备份服务器现有重要数据（如有）。|
|**主要动作**|1\. 确认当前系统状态（Ubuntu 版本、已有服务）。\&lt;br\&gt;2\. 配置腾讯云 CLS 日志服务，确保所有操作有日志可查。\&lt;br\&gt;3\. 配置腾讯云 COS 对象存储，用于关键数据远程备份。\&lt;br\&gt;4\. 检查防火墙规则，确保仅开放必要端口。|
|**产出物**|一台日志已接入 CLS、备份已配置 COS 的干净服务器。|
|**验收标准**|在腾讯云控制台能看到 CLS 日志流；COS Bucket 可正常上传测试文件。|

### 📅 阶段二：安装 Hermes Agent 并完成基础配置



**目标**：安装 Hermes，并将其配置为您的“AI 部署顾问”。利用 Hermes 的记忆系统和技能自进化能力，让它协助您完成后续所有部署步骤。



|项目|内容|
|---|---|
|**前置条件**|阶段一完成，已准备好 aiapi\.world 的 API Key。|
|**主要动作**|1\. 安装 Hermes Agent。\&lt;br\&gt;2\. 配置 Hermes 连接 aiapi\.world。\&lt;br\&gt;3\. 配置 SOUL\.md（人格）、USER\.md（用户画像）。\&lt;br\&gt;4\. 配置 MEMORY\.md 和 LESSONS\.md 初始模板。\&lt;br\&gt;5\. 配置上下文压缩参数（threshold=0\.75、protect\_last\_n=30）。\&lt;br\&gt;6\. 将 V4\.6\.2 完整方案文档导入 Hermes 作为知识库。|
|**产出物**|一个已接入大模型、已加载 V4\.6\.2 方案、已配置记忆系统的 Hermes Agent。|
|**验收标准**|与 Hermes 对话，它能准确回答 V4\.6\.2 方案中的问题，并能记住您的偏好。|



> ⚠️ **风险提示**：此阶段 Hermes 仅作为对话助手，**不能赋予任何 Shell 执行权限**。所有系统级操作需由您手动复制命令执行。
> 
>

### 📅 阶段三：利用 Hermes 引导 OpenClaw 优化



**目标**：在 Hermes 的指导下，优化您已部署的 OpenClaw，使其与 V4\.6\.2 架构对齐。



|项目|内容|
|---|---|
|**前置条件**|阶段二完成，Hermes 已可用。|
|**主要动作**|1\. 在 Hermes 引导下检查 OpenClaw 当前配置。\&lt;br\&gt;2\. 配置 OpenClaw 的 SOUL\.md、USER\.md、MEMORY\.md。\&lt;br\&gt;3\. 配置 OpenClaw 多 Agent 架构（主控/信号/风控）。\&lt;br\&gt;4\. 确认 Gateway 从 $HOME 启动（Token 成本优化）。\&lt;br\&gt;5\. 测试 OpenClaw 与 aiapi\.world 的连通性。|
|**产出物**|一个已优化、与 V4\.6\.2 架构对齐的 OpenClaw 实例。|
|**验收标准**|OpenClaw 能正常响应 Telegram 消息，多 Agent 路由配置正确。|



> ⚠️ **风险提示**：修改 OpenClaw 配置前，务必备份原始配置文件。修改后必须用 `openclaw gateway restart` 重启并验证。
> 
>

### 📅 阶段四：搭建 State Center 与核心基础设施



**目标**：部署 Redis（实时状态缓存）和 PostgreSQL（持久化数据库），以及 Grafana（监控面板）。这是整个系统的“记忆中枢”。



|项目|内容|
|---|---|
|**前置条件**|阶段三完成。|
|**主要动作**|1\. 在 Hermes 引导下创建 Docker 内部网络 `trading\_net`。\&lt;br\&gt;2\. 部署 Redis 容器并配置密码。\&lt;br\&gt;3\. 部署 PostgreSQL 容器并初始化数据库。\&lt;br\&gt;4\. 验证 OpenClaw 和 Hermes 能否连通 Redis/PostgreSQL。\&lt;br\&gt;5\. 部署 Grafana 容器并配置数据源。\&lt;br\&gt;6\. 导入预制的监控仪表盘。|
|**产出物**|运行中的 Redis、PostgreSQL、Grafana 三个容器。|
|**验收标准**|Grafana 大盘能看到数据库连接成功；Redis 能正常读写。|



> ⚠️ **风险提示**：Redis 和 PostgreSQL 的密码必须使用强密码。`\.env` 文件权限设置为 600（仅 root 可读）。
> 
> 



---



### 📅 阶段五：部署 Decision Gateway 与 Execution Engine



**目标**：部署系统的安全核心——Decision Gateway（风控裁决器）和 Execution Engine（执行引擎）。这是 **最关键的阶段**，确保 AI 无法绕过风控直接动钱。



|项目|内容|
|---|---|
|**前置条件**|阶段四完成，已申请好**币安只读 API Key** 和 **交易 API Key**（禁用提现权限）。|
|**主要动作**|1\. 在 Hermes 引导下上传 `system\_config\.json` 配置文件。\&lt;br\&gt;2\. 部署 Decision Gateway 容器。\&lt;br\&gt;3\. 部署 Market Data Service 容器（使用只读 API）。\&lt;br\&gt;4\. 部署 Execution Engine 容器（使用交易 API）。\&lt;br\&gt;5\. 配置网络隔离规则，验证 Hermes 无法访问 Execution Engine。\&lt;br\&gt;6\. 验证 Gateway 签名校验机制正常工作。|
|**产出物**|运行中的 Gateway、Market Data、Execution Engine 三个容器。|
|**验收标准**|OpenClaw 产生的模拟信号能被 Gateway 接收并裁决；Execution Engine 仅接收 Gateway 签名的订单。|



> ⚠️ **风险提示**：币安 API Key 的 **提现权限必须禁用**。交易 API Key 建议设置 IP 白名单为您的服务器 IP。此阶段 Execution Engine 建议先使用只读 API Key 进行验证。
> 
>

### 📅 阶段六：配置监控、告警与定时任务



**目标**：让系统可观测、可运维。配置关键指标的监控大屏和告警规则。



|项目|内容|
|---|---|
|**前置条件**|阶段五完成。|
|**主要动作**|1\. 在 Hermes 引导下完成 Grafana 仪表盘详细配置。\&lt;br\&gt;2\. 配置 Prometheus 告警规则。\&lt;br\&gt;3\. 配置 Telegram 告警通道（熔断通知、异常通知）。\&lt;br\&gt;4\. 配置 Cron 定时任务（每日复盘、Monte Carlo 月度模拟、记忆整理）。\&lt;br\&gt;5\. 配置 COS 自动备份脚本（数据库、配置文件）。|
|**产出物**|全功能监控面板 \+ 告警通道 \+ 定时任务体系。|
|**验收标准**|模拟触发一个告警（如手动制造滑点超限），能在 Telegram 收到通知。|



> ⚠️ **风险提示**：Cron 任务的时区必须确认与服务器一致（新加坡应为 UTC\+8）。告警不要配置得过于敏感，避免告警疲劳。
> 
>

### 📅 阶段七：分步测试与上线



**目标**：**不亏钱、不踩坑**地逐步过渡到全自动运行。此阶段不求快，求稳。



|子阶段|名称|核心配置|预计时长|关键动作|
|---|---|---|---|---|
|**7\.1**|**纸上谈兵**|`paper\_trading: true`|2\-3 周|系统跑通全流程但不实际下单。观察信号质量、服务器负载、有无死机。|
|**7\.2**|**影子学习**|`paper\_trading: false` 但 **Hermes 权重不生效**|1\-2 周|实盘跑行情，Hermes 只学不用。验证 RL 学习逻辑是否正常。|
|**7\.3**|**小额试水**|`max\_total\_portfolio\_risk: 0\.002` \(0\.2%\)|1\-2 周|**投入小额真钱**（如 100\-500 U）。重点观察滑点、成交速度、极端行情熔断。|
|**7\.4**|**正式启航**|恢复 2% 风险上限|持续运行|逐步放大资金量。保持每天看一次 Grafana 大盘的习惯。|



> ⚠️ **关键风险提示**：
> 
> 1. **阶段 7\.1 和 7\.2 绝对不能跳过**，它们是验证系统稳定性的最后防线。
> 
> 2. **阶段 7\.3 的资金量必须是你完全亏光也不会影响生活的金额**。
> 
> 3. 每个子阶段通过后，才能进入下一个子阶段。**绝不越级**。
> 
>
