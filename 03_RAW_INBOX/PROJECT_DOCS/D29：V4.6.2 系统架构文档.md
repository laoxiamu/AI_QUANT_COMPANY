```Plain Text
## 二、模块职责定义

### 2.1 模块职责边界

| 模块 | 负责什么 | **不负责什么** |
|------|---------|---------------|
| **OpenClaw Gateway** | 消息路由、Agent调度、渠道接入（Telegram/飞书/Web面板） | **不负责风控裁决、不做策略决策、不确定仓位大小** |
| **Decision Gateway** | 风控校验（熔断/风险预算/冲突检查）、资金分配（Kelly/Risk Parity）、订单签名 | **不负责信号生成、不修改策略参数、不替代任何Agent的推理能力** |
| **Strategy Engine** | 选币、环境分类、信号评分、微观结构过滤、持仓退出管理 | **不负责下单执行（只生成信号）、不管理API密钥、不直接操作数据库** |
| **SellManager** | 止损/止盈/跟踪止盈/持仓到期退出 | **不负责开仓、不决定仓位大小、不替代Risk Engine的风控判断** |
| **PostgreSQL** | 持久化存储全部信号、持仓、平仓记录 | **不负责实时缓存（Redis承担）、不参与策略决策** |
| **Redis** | 实时状态缓存（当前权益/回撤/价格/OBI） | **不是权威状态源。任何风控决策不能仅依赖Redis中的数据** |
| **systemd** | 进程保活、崩溃重启、开机自启 | **不负责进程内部的逻辑修复、不替代健康检查** |

### 2.2 关键架构原则

**原则1：职责隔离。** 策略层只输出信号，风控层只做裁决，执行层只做执行。任何一层不能跨层直接操作其他层的数据或功能。

**原则2：风控独立于策略。** Gateway的风控规则是硬编码的，任何Agent或RL模型不能修改。策略输出的信号在被执行之前必须经过Gateway的独立校验。

**原则3：权威源唯一。** 任何数据在任何时刻只有一个权威存储位置。持仓信息→PostgreSQL `signals` 表。`_positions` 字典和Redis缓存只是性能优化，不能作为决策依据。

## 三、数据流

### 3.1 标准交易请求流
```

\[1\] 行情到达 → Redis 更新价格/OBI

↓

\[2\] Strategy Engine 读取状态 → 运行策略逻辑 → 生成 Signal

↓

\[3\] Signal 发送至 Decision Gateway

↓

\[4\] Gateway 读取 PostgreSQL \+ Redis → 风控校验 → 裁决

↓

\[5\] Gateway 生成 Approved Order → 发送至 Execution Engine

↓

\[6\] Execution Engine 签名发送至交易所 → 更新 PostgreSQL

↓

\[7\] 交易平仓后，Hermes 读取结果 → 更新 RL 模型 / 写入记忆

```Plain Text

### 3.2 Ghost Position Incident 暴露的数据流断裂
```

正常路径：

PostgreSQL \(6笔活跃持仓\) → Phase 1 读取 → 检查止损 → 触发/跳过



实际路径（5月8日22:01\-23:01）：

PostgreSQL \(6笔活跃持仓\) → Phase 1 读取 \_positions \(4笔\) → 漏检2笔LONG

↑

同向覆盖已删除旧持仓

```Plain Text

**断裂点**：Phase 1 没有从 PostgreSQL 权威源读取活跃持仓列表，而是依赖已被覆盖的 `_positions` 字典。

### 3.3 权威状态源声明

| 数据 | 权威源 | 缓存/衍生 | 更新频率 | 安全约束 |
|------|--------|----------|---------|---------|
| **持仓状态** | PostgreSQL `signals` 表 | `_positions` dict | 每轮扫描写入 | Phase 1 必须从 PostgreSQL 读取，禁止仅依赖 `_positions` |
| **交易记录** | PostgreSQL `signals` 表 | — | 每笔操作写入 | 每笔交易有唯一 trade_id |
| **实时价格** | WebSocket 行情流 | Redis | 实时 | 仅用于计算参考，不作为唯一执行价格 |
| **账户权益/回撤** | Redis（由Execution Engine更新） | — | 实时 | Gateway 裁决的必读数据 |
| **策略参数** | `config.py` / `system_config.json` | MEMORY.md 文档值 | 手动修改 | 修改后必须做一致性检查（代码值 vs 文档值 vs 计划状态） |
| **Agent 行为规则** | AGENTS.md | — | 手动修改 | 在新对话中自动加载 |
```
