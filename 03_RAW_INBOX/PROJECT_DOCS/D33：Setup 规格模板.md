# D33：Setup 规格模板

## D33：SETUP\_SPEC\_TEMPLATE\.md



```Markdown
# Setup 规格模板

**文档编号**：D33
**版本**：v1.0
**创建日期**：2026-05-09
**用途**：定义 V5 及后续所有实验阶段中，每一个交易设置必须遵循的统一规格文档格式
**关联文档**：D31 RESEARCH_PROTOCOL， D30 POSITION_LIFECYCLE_SPEC， D03 Exit_System_Design


## 一、文档目的

本模板确保 V5 启动后，每一个被研究的交易设置都拥有完整、可对比、可复现的规格文档。任何 Setup 在被纳入回测或实盘验证之前，必须按照本模板完成规格定义。

V4.6.2 观察期失效的核心教训之一就是设置定义模糊——22分制评分系统没有明确的适用条件、没有清晰的退出规则、没有量化的绩效标准。本模板从根本上解决这个问题。


## 二、Setup 命名规范

### 2.1 命名格式
```

v\{主版本\}\_\{标的\}\_\{大周期方向\}\_\{核心触发\}\_\{版本号\}

```Plain Text

### 2.2 命名示例

| 命名 | 含义 |
|------|------|
| `v5_BTC_bull_sweep_v1` | V5版本，仅BTC，多头方向，sweep触发，第1版 |
| `v5_BTC_bull_fvg_v2` | V5版本，仅BTC，多头方向，FVG触发，第2版 |
| `v5_ETH_bear_choch_v1` | V5版本，仅ETH，空头方向，CHoCH触发，第1版 |
| `v5_BTC_bull_sweep_choch_fvg_v1` | V5版本，仅BTC，多头方向，sweep+CHoCH+FVG三重确认，第1版 |

### 2.3 禁止的命名方式

禁止使用 `final`、`latest`、`new`、`modified`、`test` 等无版本信息的命名。禁止使用日期替代版本号。禁止在同一版本号下反复修改参数但不更新版本号。


## 三、Setup 规格必需字段

### 3.1 基本信息

| 字段 | 说明 | 示例 |
|------|------|------|
| **Setup ID** | 唯一标识符 | `v5_BTC_bull_sweep_v1` |
| **创建日期** | YYYY-MM-DD | `2026-05-15` |
| **状态** | DRAFT / BACKTEST / PAPER / LIVE / FROZEN | `BACKTEST` |
| **作者** | 设计者 | — |
| **简短描述** | 一句话说明此Setup的交易逻辑 | `BTC在4H多头背景下，15m流动性扫低后反转做多` |

### 3.2 Regime 条件

| 字段 | 说明 | 示例 |
|------|------|------|
| **大周期方向** | 日线/4H的趋势方向判定 | `日线 MA20斜率>0 + 价格在MA20上方 + 高低点递增` |
| **市场状态** | 允许此Setup运行的 environment | `仅Bullish Regime` |
| **禁止状态** | 明确禁止的状态 | `Sideways、Bearish、宏观事件窗、周末` |
| **Regime确认K线数量** | 用于确认方向的K线根数 | `日线=5根确认趋势` |
| **Regime失效条件** | 什么情况下退出此Regime | `日线收盘价跌破MA20且MA20斜率转负` |

### 3.3 Trigger 条件

| 字段 | 说明 | 示例 |
|------|------|------|
| **核心触发事件** | 什么事件发生后Setup被激活 | `Liquidity Sweep: 15m K线低点跌破前3根K线低点后收回` |
| **触发确认条件** | 如何确认触发是有效的 | `sweep K线收盘价在sweep前低点上方 + 下一根K线不创新低` |
| **触发K线周期** | 在哪个时间级别上检测触发 | `15m K线` |
| **触发失效时间** | 触发后多长时间内未确认则放弃 | `sweep发生后4根K线内未出现确认信号则放弃` |
| **并发限制** | 同一时间最多允许几个此Setup的活跃持仓 | `1个（同币种同方向只能有一个活跃持仓）` |

### 3.4 Invalidation 条件

| 字段 | 说明 | 示例 |
|------|------|------|
| **结构失效条件** | 什么情况下入场逻辑不再成立 | `价格跌破sweep低点（即sweep失败）` |
| **Regime失效条件** | 什么情况下大周期背景改变 | `日线MA20斜率转为负值` |
| **Time Invalidation** | 入场后多长时间无进展视为失效 | `入场后8根1H K线（8小时）未触及止盈线且未确认趋势延续` |

### 3.5 Stop 规则

| 字段 | 说明 | 示例 |
|------|------|------|
| **止损方式** | Structure Stop / Fixed / ATR | `Structure Stop: swing_low - 0.5×ATR(14) on 1H` |
| **止损计算参数** | 具体参数值 | `swing_low = 最近3根1H K线的最低价` |
| **止损缓冲** | ATR或固定点 | `0.5×ATR(14)` |
| **止损最大距离** | 止损线距离入场价的最大比例 | `≤ 入场价的2%` |
| **止损调整规则** | 是否随着持仓推移调整止损位置 | `不调整。结构止损在入场时设定后固定不变` |

### 3.6 Exit 规则

| 字段 | 说明 | 示例 |
|------|------|------|
| **止盈方式** | 固定/跟踪/结构 | `跟踪止盈: 浮盈≥2%激活，从peak_pnl回撤0.5%退出` |
| **时间退出** | 持仓超过多长时间无条件退出 | `48根1H K线（48小时）` |
| **结构退出** | 什么结构变化触发退出 | `价格跌破swing_low入场后形成的第一个higher_low` |
| **退出优先级** | 多个退出条件触发时的顺序 | `止损 > 结构退出 > 时间退出 > 跟踪止盈` |
| **部分平仓** | 是否分批退出 | `无。全额平仓` |

### 3.7 仓位规则

| 字段 | 说明 | 示例 |
|------|------|------|
| **仓位计算方式** | 固定/ATR动态/Kelly | `ATR动态: position∝1/ATR` |
| **最大风险暴露** | 单笔最大亏损占账户比例 | `≤ 账户余额的1%` |
| **环境权重** | 不同市场状态下的仓位调整 | `Bullish Regime = 1.0×; Transition Regime = 0.5×` |
| **连亏缩仓** | 连续亏损后的自动缩仓 | `连续亏损2笔→仓位系数×0.5` |

### 3.8 Metrics 指标

| 字段 | 说明 | 示例目标值 |
|------|------|-----------|
| **期望值** | 胜率×平均盈利 + 败率×平均亏损 | `Expectancy > 1.0` |
| **盈亏比** | 平均盈利/平均亏损 | `RR > 1.5` |
| **最大回撤** | 权益峰值到谷值的最大跌幅 | `MaxDD < 20%` |
| **夏普比率** | 收益率/波动率 | `Sharpe > 0.5` |

### 3.9 Sample Requirements 样本要求

| 字段 | 说明 | 示例 |
|------|------|------|
| **最少交易次数** | 回测验证的最小样本量 | `≥ 1000次穿仓` |
| **最少盈利笔数** | 盈利交易的最小数量 | `≥ 100笔（确保PnL分布可评估）` |
| **数据时间范围** | 回测使用哪个时间段的数据 | `2024-01-01 至 2026-04-30` |
| **训练/验证集划分** | Walk-forward的划分方式 | `训练集=2024年全年，验证集=2025年全年，测试集=2026年1-4月` |
| **是否排除特殊事件** | 是否排除极端行情日 | `是。排除CPI/FOMC日及前后1小时。排除交易所宕机日` |

## 五、Setup 生命周期管理



### 5\.1 Setup 状态流转



```Plain Text
DRAFT → BACKTEST → PAPER → LIVE → FROZEN
```



|状态|含义|进入条件|
|---|---|---|
|**DRAFT**|规格已定义，尚未开始回测|Setup Spec文档完成|
|**BACKTEST**|正在历史数据上验证|回测脚本完成，数据准备就绪|
|**PAPER**|Paper Trading验证中|回测Expectancy\&gt;1\.0且Sharpe\&gt;0\.5|
|**LIVE**|小资金实盘验证|Paper Trading连续20笔平仓Expectancy\&gt;1\.0|
|**FROZEN**|已归档，不再运行|被新版本替代或验证失败|



### 5\.2 Setup 版本迭代规则



- 每次修改Setup中的任何参数（Stop/Exit/仓位/Regime条件），必须创建新版本号

- 新版本Setup从DRAFT状态开始，不继承旧版本的回测数据

- 旧版本归档为FROZEN，保留所有回测数据和评估报告

- 版本号递增规则：v1→v2→v3\.\.\.（不跳号，不重用已归档的版本号）

    

### 5\.3 Setup 废弃条件



一个Setup在以下情况下被废弃进入FROZEN状态：

- 回测验证Expectancy\&lt;1\.0且连续2个版本均未达标

- PAPER阶段连续20笔平仓Expectancy\&lt;1\.0

- 市场结构发生根本变化导致Setup的Regime条件不再出现

- 被新版本替代

    

    

## 六、与V4 Baseline的对比要求



任何V5 Setup在BACKTEST阶段必须包含与V4 Baseline的对比：



|对比指标|说明|
|---|---|
|**Expectancy**|V5 Setup vs V4 Baseline在同一时间段内的期望值对比|
|**MaxDD**|V5 Setup的最大回撤是否低于V4 Baseline|
|**Sharpe**|V5 Setup的夏普比率是否高于V4 Baseline|
|**交易频率**|V5 Setup的信号密度是否合理（不能过少导致样本不足，不能过多导致过拟合）|



对比报告作为Setup回测报告的独立章节。如果V5 Setup在所有指标上均无改善，此Setup应被废弃。





**本文档是V5启动后的核心规范之一。任何Setup在进入回测验证之前，必须先按此模板完成规格定义并归档保存。未经规格定义的Setup不允许进入BACKTEST阶段。**
