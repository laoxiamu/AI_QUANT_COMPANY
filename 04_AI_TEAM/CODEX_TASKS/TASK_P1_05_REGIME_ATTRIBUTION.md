# TASK-P1-05 市场状态框架 + 跨信号归因（研究第一步）

**创建：** 2026-06-06 · Claude（CTO）· 执行人：Codex
**依据：** DEC-057 研究顺序铁律；DEC-058 重排队列（本任务取代 P1-04/P1-03 的优先级）
**性质：** **诊断研究**——不计失败次数、不碰 Holdout、不消耗独立 Alpha 配额。目的：建立市场状态框架并定位 edge 活在哪个状态/信号。

> 执行前请先按协作规范输出"专业审查七问"。若你认为状态定义或归因口径有更优方案，先在报告提异议再落地。

## 1. 目标（铁律前三步）

1. **市场状态划分**：用标准、独立的分类器把 BTC/ETH/SOL 4H 历史划分为 `趋势↑ / 趋势↓ / 震荡`。
2. **各状态收益/回撤归因**：把现有信号的表现按状态切开。
3. **产出"信号 × 状态 edge 地图"**：明确哪个信号在哪个状态有正且有意义的 edge。

## 2. 状态分类器（两轴，标准值，多镜头交叉，防过拟合）

**P1-04 的关键教训（DEC-060）：ADX 只测"趋势强度"，不分"牛熊方向"——2022 熊市反弹被误判为上升趋势→做多 -23.72%。
因此状态必须显式拆成两个独立轴：**

- **轴1 强度**：ADX(14) on 4H（>25 趋势 / <20 震荡 / 20~25 迟滞）；Kaufman 效率比(ER,60) 作第二镜头交叉校验，报告一致率。
- **轴2 牛熊方向**：高周期方向判定（如日线 close vs 200日MA、或日线 MA 斜率/高低点结构）——**这是 P1-04 缺的轴**。
- 组合为状态网格：{趋势↑·牛 / 趋势↑·熊(反弹陷阱) / 趋势↓·牛 / 趋势↓·熊 / 震荡·牛 / 震荡·熊}。
- 所有分类仅用 ≤t 数据（无前视）。报告各状态时间占比、典型时长。

**★ 必答：用两轴状态归因 P1-04 的 392 笔交易，确认 2022 -23.72% 是否几乎全部落在"趋势↑·熊"格（即熊市反弹被强度轴误判）。** 这直接验证"加一个高周期方向轴能否消除 2022 反证"。

## 3. 归因对象（用现有产物，勿重跑信号）

A. **TSMOM v1 交易**（`output/p1_01_tsmom_trades.csv`）：按入场状态切分 → 每状态 笔数/净P&L/胜率/平均R/MaxDD。
B. **Sweep 家族事件**（`output/events_v5_sweep_*.csv` 等）：按事件发生时状态切分，统计 t+6/t+12/t+24 前向收益分布与显著性。
   - ★ 重点检验假设：**Sweep（逆势/均值回归型）是否在"震荡"状态才有正 edge**，而我们当年只在 EMA200 多头 regime 测、把它在全状态混测后判失败 → 可能误杀。
C. 如可行，TSMOM v2 交易亦按状态切分对照。

## 4. 输出（核查/诊断报告，非通过/失败判定）

- 三态时间占比 + 两分类器一致率。
- 表：信号 × 状态 → 笔数/事件数、净P&L、胜率、平均R、MaxDD、（事件类）前向收益与p值。
- **edge 地图结论**：哪个信号在哪个状态有正且稳健 edge；哪些是死重。
- 明确：Sweep 家族在震荡态是否被误杀（是/否 + 数据）。
- 对照 DEC-018 给各"信号×状态"格子的样本档级。
- 不做仓位/成本优化（那是后续第5步）。
