# MEMORY_EXTRACTION_PROTOCOL.md

**版本：** 1.0  
**创建时间：** 2026-06-01  
**阶段：** Phase 0A · AI能力基础设施建设  
**状态：** ACTIVE  
**适用范围：** 所有历史文档读取、信息提炼、归档、写入 MEMORY_CORE 的全部操作

---

## 1. 信息等级定义

本协议定义七个信息等级。所有进入 MEMORY_CORE 的信息必须明确标注等级，未标注等级的信息一律视为 UNKNOWN，不得使用。

---

### DECISION（已确认决策）

**定义：**  
经用户在对话中明确确认、或已写入 DECISION_LOG.md 的架构决策、方向选择、约束条件。

**判断标准：**  
- 用户已明确说"确认"、"采用"、"决定"，或等价表述
- 已在 DECISION_LOG.md 中存在记录

**允许来源：**  
- 用户当前对话的明确确认
- DECISION_LOG.md 已有记录

**允许进入 MEMORY_CORE：** ✅ 是。写入 DECISION_LOG.md。

---

### FACT（已验证事实）

**定义：**  
已通过实际操作或工具验证的客观事实，例如文件是否存在、环境版本号、命令执行结果。

**判断标准：**  
- 有工具执行结果支撑（如 Desktop Commander 返回的文件列表、命令输出）
- 可重复验证，不依赖主观判断

**允许来源：**  
- Desktop Commander 工具执行结果
- 当前对话中实际验证的操作结果

**允许进入 MEMORY_CORE：** ✅ 是。写入 CURRENT_STATE.md 或 PROJECT_CONTEXT.md。

---

### EXPERIENCE（经验观察）

**定义：**  
历史操作中积累的经验性观察，有一定参考价值，但未经当前项目实际验证，不具备直接可信度。

**判断标准：**  
- 来自历史对话记录、过往版本文档
- 描述的是"曾经发生的事"或"过去认为有效的做法"
- 尚未在当前项目环境中验证

**允许来源：**  
- 03_RAW_INBOX 中的历史文档（仅作参考）
- ChatGPT/DeepSeek/Claude 历史对话导出

**允许进入 MEMORY_CORE：** ⚠️ 有条件。须明确标注来源与等级，仅作参考，不得作为架构依据。写入前须用户确认。

---

### HYPOTHESIS（待验证假设）

**定义：**  
尚未验证的推测、方向假设、未经确认的方案建议。包括 Claude 在对话中的推演和分析。

**判断标准：**  
- 无实际验证支撑
- 来源为推理、类比、建议，而非执行结果
- 历史文档中未注明验证状态的内容

**允许来源：**  
- 任何未经验证的来源
- Claude 在对话中的分析与推演（默认等级）

**允许进入 MEMORY_CORE：** ❌ 否。不得写入 MEMORY_CORE，仅允许保留在 03_RAW_INBOX 或 99_TEMP。

---

### DEPRECATED（已废弃）

**定义：**  
曾经有效但已被明确替代、推翻或过期的信息。包括旧版本架构、已放弃的方案、已失效的参数。

**判断标准：**  
- 有后续决策明确替代它
- 用户明确标注废弃
- 时间戳明显早于当前阶段且无延续性说明

**允许来源：**  
- 历史文档（仅用于识别，不用于参考）

**允许进入 MEMORY_CORE：** ❌ 否。不得写入。如需保留历史记录，标注 DEPRECATED 后归入 03_RAW_INBOX。

---

### CONFLICT（冲突信息）

**定义：**  
来自不同来源的信息，就同一事项存在相互矛盾或无法调和的内容。

**判断标准：**  
- 两条或以上信息描述同一主题，但结论不同
- 无法通过时间顺序或来源优先级自然裁决

**允许来源：**  
- 任何来源均可触发 CONFLICT 标记

**允许进入 MEMORY_CORE：** ⛔ 暂停。必须先解决冲突，等待用户裁决后方可写入。冲突记录本身可临时存入 99_TEMP。

---

### UNKNOWN（信息缺失）

**定义：**  
所需信息不存在、来源不明、或现有资料不足以支撑判断。

**判断标准：**  
- 资料中未提及
- 来源不可追溯
- 无法归入以上任何等级

**允许来源：**  
- 任何来源均可触发 UNKNOWN 标记

**允许进入 MEMORY_CORE：** ❌ 否。必须明确标注 UNKNOWN，等待后续补充，禁止推断填补。

## 2. 历史文档读取规则

Claude 读取 03_RAW_INBOX 中任何历史文档时，以下规则强制执行，无例外。

### 2.1 禁止自动继承

| 禁止行为 | 说明 |
|----------|------|
| 禁止自动继承历史架构 | 历史文档中描述的系统架构不得直接作为当前架构依据 |
| 禁止自动继承历史结论 | 历史对话中得出的结论不得视为已在当前项目验证 |
| 禁止自动继承历史参数 | 历史文档中的具体数值、配置参数须重新验证后方可使用 |
| 禁止将聊天记录视为事实 | ChatGPT/DeepSeek/Claude 历史对话默认等级为 EXPERIENCE 或 HYPOTHESIS |
| 禁止将单一文档视为最终依据 | 任何单一来源的信息须交叉核实或用户确认后方可升级等级 |

### 2.2 读取时强制操作

每次读取历史文档，Claude 必须：

1. **标注来源**：记录文件名、所在目录、文档类型
2. **标注初始等级**：未注明来源可信度的内容，默认为 HYPOTHESIS
3. **识别时效性**：注明该文档对应的历史阶段（如可识别），标注是否可能已过期
4. **不得跨文档合并推断**：不得将多份历史文档的内容合并后推断出新结论，除非用户明确指示

## 3. MEMORY_CORE 写入规则

### 3.1 各文件允许写入的内容

**PROJECT_CONTEXT.md**
- ✅ 允许：已验证的项目背景信息（FACT 级别）
- ✅ 允许：用户明确确认的项目范围与目标描述
- ❌ 禁止：推测性内容、历史文档中未经验证的目标描述

**CURRENT_STATE.md**
- ✅ 允许：工具实际验证的当前环境状态（FACT 级别）
- ✅ 允许：已完成任务的记录（含完成时间）
- ❌ 禁止：计划中但未完成的内容作为"当前状态"写入

**DECISION_LOG.md**
- ✅ 允许：用户明确确认的决策（DECISION 级别）
- ✅ 允许：每条决策须包含：决策内容、时间、来源、影响范围
- ❌ 禁止：Claude 的建议或推演，未经用户确认不得写入
- ❌ 禁止：历史文档中的"历史决策"直接写入，须用户逐条确认

**SYSTEM_RULES.md**
- ✅ 允许：用户明确确认的操作规则和约束
- ✅ 允许：本协议及 Instructions 中明确规定的规则
- ❌ 禁止：Claude 自行推断的规则
- ❌ 禁止：未标注来源的规则条目

### 3.2 全局禁止写入 MEMORY_CORE 的内容

- HYPOTHESIS 级别的内容
- DEPRECATED 级别的内容
- UNKNOWN 级别的内容（未解决前）
- CONFLICT 状态下的任意一方（冲突解决前）
- 对话中的临时分析、推演草稿
- 未注明等级和来源的任何信息

## 4. 冲突处理协议

### 4.1 冲突触发条件

当 Claude 在读取或提炼过程中发现以下情况，必须触发 CONFLICT 流程：

- 两份文档就同一决策描述不同结果
- 同一参数在不同来源有不同数值
- 历史文档中的架构描述与已确认决策矛盾

### 4.2 冲突处理步骤（强制执行）

1. **立即停止当前提炼工作**，不得继续推进
2. **标记 CONFLICT**，说明：
   - 冲突主题
   - 来源 A（文件名 + 位置 + 内容摘要）
   - 来源 B（文件名 + 位置 + 内容摘要）
   - 冲突性质（数值冲突 / 结论冲突 / 架构冲突）
3. **保留原始来源**，不得修改或删除任何一方
4. **不允许 Claude 自行裁决**，即使 Claude 认为某一方明显更合理
5. **等待用户明确指令**，告知采用哪方或提供新决策
6. 用户裁决后，被废弃的一方标注 DEPRECATED，采用方升级为 DECISION 或 FACT

### 4.3 冲突记录格式

```
[CONFLICT]
主题：
来源A：文件名 | 章节 | 内容摘要
来源B：文件名 | 章节 | 内容摘要
冲突性质：
等待用户裁决：是
```

## 5. 信息缺失处理协议

### 5.1 缺失触发条件

- 所需信息在所有可用文档中均未提及
- 来源文档损坏、不完整或无法读取
- 信息存在但来源不可追溯

### 5.2 缺失处理步骤（强制执行）

1. **标记 UNKNOWN**
2. **明确说明**：缺失的是什么信息、影响哪个模块或决策
3. **禁止推断补全**：Claude 不得用"合理推断"填补缺失信息
4. **禁止降级使用**：不得将 UNKNOWN 信息以 HYPOTHESIS 身份进入后续分析
5. **等待用户指示**：提供补充信息或决定如何处理缺失

### 5.3 缺失记录格式

```
[UNKNOWN]
缺失信息：
影响范围：
已查阅来源：
补充建议（仅列出建议方向，不填补内容）：
```

## 6. 输出规范

所有历史资料提炼结果，无论最终写入哪个文件，必须使用以下统一格式。

### 6.1 标准提炼条目格式

```
---
信息等级：[DECISION / FACT / EXPERIENCE / HYPOTHESIS / DEPRECATED / CONFLICT / UNKNOWN]
来源文件：[文件名]
来源目录：[03_RAW_INBOX/子目录]
来源位置：[章节名称 / 段落序号 / 页码（如可识别）]
提炼内容：
  [提炼后的简洁描述，不得直接复制粘贴原文，须用项目语境重新表述]
可信度说明：
  [说明为何归入此等级，有何不确定性]
处理状态：[待用户确认 / 已确认写入 / 已标记废弃 / 冲突待裁决]
---
```

### 6.2 批次提炼记录

每次提炼工作结束，须在 99_TEMP 生成一份批次摘要，包含：

- 本次提炼涉及的文件列表
- 各等级信息数量统计
- 发现的 CONFLICT 数量及主题
- 发现的 UNKNOWN 数量及主题
- 建议写入 MEMORY_CORE 的条目列表（等待用户确认）

### 6.3 禁止的输出行为

- 禁止直接复制原文作为提炼内容
- 禁止在未标注来源的情况下输出任何提炼条目
- 禁止将多个来源的内容合并后以单一条目呈现（须分条保留各自来源）
- 禁止在用户未确认前将提炼结果直接写入 MEMORY_CORE

---

*本协议为 Phase 0A 阶段信息治理基础文件，适用于后续所有历史文档提炼工作。协议本身的修改须经用户明确确认后方可执行。*
