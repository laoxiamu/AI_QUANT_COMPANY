## 2. AI 组织与决策工作流

**角色（同一模型戴多帽，但上下文/任务/验收责任隔离）：**

- **Claude 主会话 = CEO/CTO：** 综合裁决、战略、投研主导、风控判断、对 Codex 验收。
- **子会话（隔离）：**
  - **Research Architect**：Alpha 机制与研究路线设计、命题与预登记起草。
  - **Risk Reviewer（独立否决权）**：对任何策略/定仓做爆仓概率/相关性/生存审查，可否决。
  - **Product / Operations**：产品与流程。
  - **Independent Review**：不继承提案者结论的独立复核（重算、Deflated Sharpe、红队）。
- **Codex 实现会话：** 代码、测试、复现。

**关键方案决策流程：**
```
主责起草(R) → 独立角色反审(Risk Reviewer / Independent Review,可否决)
   → 主责修订 → CTO 综合裁决 → Founder 仅批 D 级
```

**交接（自动化）：** Claude 经 Desktop Commander **直调 Codex CLI**（DEC-061）去掉 Founder 中间人；文件式 handoff + 标准模板。
> ⚠️ **诚实标注：** 该直调通道是"决策已定、尚未接通测试"。它是"放手长跑"的关键开关，**必须在 R1/最小验证里真正接好并验证**，不能停在"文档已写"。

**何时可放手（Founder 视角）：** 公司 OS 冻结 + 跑通一次最小验证 + 直调通道验证之后。届时你只看日报、被 ping 只在 D 级，L1/L2/L3 审计当护栏。即便如此仍是"稳定流程里的可控转向"，方向级仍上升给你。
