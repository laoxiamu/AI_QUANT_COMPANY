## 0. 概念定调：部门 = 帽子，不是多 AI

"公司多部门"是**职责抽象**，真实执行者只有三个：

- **Founder（你）**：董事会。只管 D 级（钱/阶段/范围/重大架构），日常看日报。
- **Claude（我）**：戴判断型帽子——主理人/CTO、产品、项目治理、投研主导、风控判断、市场调研、验收。

"各司其职"= 帽子清晰 + 交付物标准化 + 谁验收明确 + 何时上升 Founder 明确。这四条做到，才有"少卡壳、丝滑、长跑"（§7）。

## 2. 责任层：谁出、谁验、谁拍（RACI）

**R**=产出 **V**=验收/挑战 **D**=拍板。原则：**产出与验收分离且交叉**。

| 交付物 | R | V | D |
|---|---|---|---|
| 产品方案 | Claude | Codex（可行性）+Founder | Founder |
| 技术方案 | Claude 设计→Codex 细化 | Claude↔Codex 互审 | Founder（架构级）|
| 项目/治理方案 | Claude | Codex（独立审计 L1）+Opus（L2）| Founder |
| 投研方案 | Claude | Codex 核查 | Founder（方向）|
| 风控方案 | Claude | Codex | Founder（风格档/资金）|
| 市场/合规调研 | Claude（搜网）| Codex/Founder | Founder（范围）|
| 数据/技术实现 | Codex | Claude 验收 | —（除非超范围）|
| QA/验证 | Codex+Claude 交叉 | 互为 V | — |

> 6-07 三份审计（Claude/Codex/Opus 交叉）就是这套交叉验收的实例，证明三视角比单方自评可靠。

## 3. 工作流：方案怎么出（串/并、产出/验收）

**单份方案标准流水线：**

```
立项(七问:值不值得做?) → 起草(R) → 挑战/验收(V:七问+红队)
  → D级? ──是→ 上升 Founder 拍板
          └─否→ CTO 直接定 → 执行 → 验收 → 归档(更新 CURRENT_STATE)
```

**规则：**
- **七问前置**（DEC-057）：起草前先答"验证什么机制/证据是否支持/有无更上游问题"，挡掉不值得做的。
- **串行**：有依赖的（论点 → 技术架构 → 建设；数据层 → 机制验证 → 策略回测）。
- **并行**：相互独立的（风控方案 ∥ 数据方案 ∥ 市场/合规调研）。
- **一人出一人验**为默认；**多视角讨论**只留给高风险分叉项（方向选择、架构冻结、止损口径）。开会是成本，不滥用。
- **方案修订频率**：地基方案冻结后只在阶段切换或重大决策变更时改；日常是参数微调与执行，不推倒重来——对应你要的"中间会优化但不频繁"。

## 5. 系统运行流程（实盘态，目标预览）

```
行情数据 → 信号生成(机制触发) → 风控&定仓(认亏%+止损距离→仓位/杠杆)
  → 下单执行 → 持仓入库(DB唯一权威) → 对账(每N分钟比对交易所)
  → 监控告警(异常推送+急停) → 绩效记录 → 定期复盘 → 改进回流
```

现在画出来是给技术方案一个靶子，不是现在建。两条硬要求来自 V4 事故：**持仓以 DB 为权威（非内存）+ 必须有对账模块**。

## 7. 执行引擎：Claude/Codex 分工 + 子会话/子 agent（"丝滑长跑"怎么实现）

**两层分工：**
- **角色分工**：判断型 → Claude；实现型 → Codex。清晰不细碎。
- **执行隔离**：不同职能用**独立会话/任务**承载（DEC-013），上下文短、便宜、不串味；靠**文件式 handoff**当共享记忆。

**子会话 vs 子 agent：**
- **子会话（主力）**：每职能开独立对话（投研设计 / Codex 实现 / 验收 / 治理各自独立）。简单、可控、留痕、低成本。
- **子 agent（点用）**：仅"并行扇出"的有界任务才用（并行调研多猎场、独立验收）。
- **不上**：重型多 agent 编排框架（LangGraph/A2A 等）= Phase 2 再评估。现在上 = 风险 B（只建系统不验证）。

**丝滑真正靠这五样（不是靠更多 AI）：**
1. 论点冻结且正确（Thesis v1 已做）——错论点才导致每对话返工。
2. 交付物模板标准化（本文 §1）——handoff 干净，接手不用猜。
3. 决策闸预定义——日常按门槛走，只有真 D 级才停下等你。
4. 状态文件单一权威（CURRENT_STATE）+ 启动协议——冷启动快速恢复、不偏航。
5. 新对话"主理人开局"——我主动牵头，不等你一步步引导。

> 第 5 条要写进 CLAUDE.md 才能保证每个对话都触发（见 §9）。
