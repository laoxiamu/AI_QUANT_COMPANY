## 一、本协议的目的

本协议定义 V5 所有策略研究、因子验证、假设测试的标准流程。

V4 观察期的核心教训之一：研究协议从未存在——没有实验隔离规则、没有统计有效性标准、13天内修改了7个核心变量，导致无法追溯任何改动的独立贡献。本协议是防止历史重演的基础方法论。

---

## 二、核心原则（铁律，不可绕过）

1. **单变量原则**：一次只验证一个变量。发现新问题，先记录，完成当前实验后再处理。
2. **预登记原则**：假设在回测开始前必须完整定义并记录，不允许在看到数据后修改假设。
3. **独立验证原则**：Walk-Forward 是硬门槛，不可跳过，不允许只做样本内测试。
4. **成本完整性原则**：所有回测必须包含完整成本建模，不允许使用理想化成本。
5. **结论诚实原则**：无论结果好坏，必须输出明确结论并归档。模糊表述（「有一定效果」）不被接受。

## 四、数据准备规范

### 4.1 数据版本控制

每次实验使用固定的数据快照，实验期间不更新数据。数据文件标注版本号和日期范围。

```
数据文件：BTC_USDT_4H_[起始日期]_[结束日期].csv
当前文件：BTC_USDT_4H.csv（2019-01-01 至 2025-12-31）
```

每次研究前检查数据是否需要更新（上次更新时间 + 数据截止日），更新后重新记录数据版本。

### 4.2 数据三分（强制）

```
训练集（Train）：  60%   → 参数初步探索，不用于最终评估
验证集（Val）：    20%   → 参数调优后的初步验证
Holdout集：       20%   → 全程封存，仅最终确认时使用一次，用即作废，不可复用
```

**Holdout 封存规则：** Holdout 集的数据范围在实验开始时写入假设文档，且在 Walk-Forward 通过前不得查看。任何在 Holdout 上的测试都是一次性的，用完即宣告该假设的验证周期结束。

### 4.3 统计污染样本排除

以下交易记录从所有统计中排除，并在报告中单独标注数量和原因：
- `trade_status = INVALIDATED`（生命周期断裂）
- `trade_status = ORPHANED`（未被检查的孤儿持仓）
- 数据记录不完整的条目

## 五、回测执行规范

### 5.1 成本建模（强制，不得简化）

所有回测必须包含以下三项成本，缺少任一项为不合格回测：

| 成本项 | 参数 | 说明 |
|--------|------|------|
| 手续费 | 0.1%（单边）| Maker/Taker 取保守值 |
| 滑点 | 0.05%-0.1%（保守取0.1%）| 市价单冲击成本 |
| 资金费率（Funding）| 按历史费率或保守常数（DEC-019）| U本位永续每8小时结算，必须建模 |

### 5.2 Walk-Forward 验证（硬门槛）

**不允许只做样本内回测。** Walk-Forward 的目的是验证策略在未见过的数据上是否仍然有效。

执行方式：
```
将历史数据按时间滚动切分：
  窗口1：训练[T0-T1] → 验证[T1-T2]
  窗口2：训练[T1-T2] → 验证[T2-T3]
  窗口3：训练[T2-T3] → 验证[T3-T4]
  ...（至少3个窗口）
  
要求：每个验证窗口的 Sharpe > 0（不要求全部通过，但要分析失败窗口的原因）
最终 Walk-Forward 通过标准：多数窗口验证集 Sharpe > 0，且趋势无明显衰减
```

### 5.3 杠杆敏感性测试（第一次用1x后执行）

使用 1x 基础回测确认信号纯度后，做杠杆敏感性对比：
- 测试范围：1x / 2x / 3x
- 关注指标：Sharpe 变化、最大回撤变化、强平风险距
- 输出：杠杆选择建议（DEC-019 要求：杠杆上限为开放参数，Phase 3 实盘前由 Founder 设定，建议初期 ≤3x）

## 六、评估标准（验证门槛）

### 6.1 假设验证门槛（DEC-018，统一标准）

| 指标 | 要求 | 说明 |
|------|------|------|
| Expectancy | > 1.0 | 扣完全部成本后正期望 |
| Sharpe Ratio | > 1.0 | 风险调整收益（1x杠杆） |
| Max Drawdown | < 25% | 风险可控 |
| 历史触发次数 | ≥500 = 确认级 / 300-499 = 探索级 | 双层门槛（DEC-018）|
| IC（因子研究时）| > 0.02（Spearman）| 有统计意义的预测力 |
| Walk-Forward | 必须通过 | 不可只做样本内验证 |
| 多时间段稳定性 | 牛/熊/震荡均表现正向 | 非单一时期特有 |

### 6.2 档级判断（对照 DEC-018）

- **确认级（≥500次）**：可作进实盘候选，进入 Phase 2 系统建设评估
- **探索级（300-499次）**：初步结论有效，但**禁止进实盘**，必须并品种或降周期扩样本后复测升级
- **不成立（<300次）**：换 Setup 或并品种，不在此基础上继续优化

### 6.3 IC 计算标准

- 方法：Spearman 秩相关
- 窗口：30日滚动窗口
- 多重检验校正：Benjamini-Hochberg 方法
- 报告内容：IC均值 + IC标准差 + IC正负率 + IR（IC/ICIR）

## 七、Red Team 自查清单（提案人强制执行）

在输出任何正面结论之前，必须逐项回答以下问题：

```
□ 单变量确认：本次只测试了一个变量，没有同时改变其他参数？
□ 前视偏差检查：信号判定时使用的所有数据，在实际交易时刻是否都已知晓（无未来数据）？
□ 样本外测试：Walk-Forward 是否通过？Holdout 还未被使用？
□ 参数敏感度：关键参数±20%后，结论是否仍然成立？
□ 多重检验校正：如果测试了多个参数组合，是否做了多重检验校正？
□ 数据挖掘偏差：这个假设是在看到数据后才定义的，还是预先登记的？
□ 成本完整性：回测成本是否包含手续费+滑点+资金费率？
□ 时期依赖性：结论是否只在特定时期（如牛市）成立？多周期验证结果如何？
□ 样本量充分性：触发次数是否符合 DEC-018 的档级要求？
□ 幸存者偏差：回测中是否使用了当时不可知的信息（如币种是否后来退市）？
```

**每一项打勾表示已检查且通过，任何未打勾项必须在结论报告中明确说明。**

## 十、方法论借鉴落地（v1.1 新增，DEC-056/060/061）

> 原则（DEC-056）：不自己发明 → 先学 → 借鉴 → 集成 → **最后才重写**。本节把工具调研（DEC-059）经主对话审阅后采纳的方法论固化为研究纪律。库的引入纪律：**优先自实现可审计的小函数（≤50行），借鉴成熟开源实现作参考，不轻易引入运行时硬依赖；确需第三方库先做安全审计（pip show + 源码 grep）并 pin 版本。**

### 10.1 退出框架：三屏障（Triple Barrier）为方向性事件策略的默认基准

**适用边界（DEC-061，Codex 评审②）：三屏障是"方向性事件策略"（如 Sweep、突破、动量入场）的默认基准退出框架，不是所有机制的通用默认。** carry / 资金费率 / 做市 / 跨市场套利等非方向性机制，退出应由**机制失效条件**决定（如基差收敛、价差回归、对冲腿失衡），**预登记中必须说明为何不适用三屏障**。

对适用的方向性事件策略：

- **上轨（止盈）+ 下轨（止损）+ 垂直轨（时间）**，任一触发即退出。
- 上下轨基于波动率动态设置（滚动 ATR；对应 0B16 诊断 ATR×1.9 结论），不用固定百分比。
- 垂直轨 = 最大持仓 K 线数（如 t+24）。
- 实现：自写 `triple_barrier_labels.py`（≤50行），借鉴 López de Prado / MLFinPy 实现作参考，**不引入 MLFinPy 硬依赖**（DEC-061）。
- 输出每笔：label(+1/-1/0) + 实际退出时间 + 触发屏障类型。
- 首次接入只替换退出逻辑、不动入场信号（单变量原则），并预登记为新实验版本。

### 10.2 规则式状态门控（Rule-based State Gating）

Phase 1 状态过滤的**正式名称**。主信号（如 TSMOM 方向 / Sweep 事件）产生候选，按**规则式状态判断**（§10.3 两轴）决定"这次做不做"，**不改方向**。

- ⚠️ **命名纪律（Codex 评审③）：本阶段不称"Meta-Labeling"。** 严格 Meta-Labeling 含独立训练的二级模型；"Meta-Labeling"在此仅作方法论来源借鉴。
- 当前阶段为**规则式门控**，**不在 Phase 1 训练二级 ML 模型**。
- 报告指标用**覆盖率（通过门控的信号占比）/ 保留率 / 状态间收益差异**等描述性指标，**不要求 精确率/召回率**——后者基于事后盈利标签，在规则式门控里会诱导按结果反调过滤器、重新引入过拟合。
- **只有在真正训练独立二级模型时**，才强制 精确率/召回率 + Purged/Embargo CV（§10.4）。

### 10.3 市场状态分类标准（两轴，强制；与 TASK_P1_05/DEC-060 一致）

**铁律：状态必须显式拆成两个独立的轴，缺一不可。** P1-04 的 2022 -23.72% 根因正是只用了强度轴、把熊市反弹误判为上升趋势（DEC-060）。

- **轴1 强度**：ADX(14)（>25 趋势 / <20 震荡 / 20~25 迟滞）；Kaufman 效率比(ER) 作第二镜头交叉校验，报告两者一致率。
- **轴2 牛熊方向**：**独立的高周期方向判定**（如日线 close vs 200日MA、或日线 MA 斜率/高低点结构）。
- ⚠️ **禁止**用同周期 DI+/DI−（或任何与强度轴同源、同周期的指标）冒充方向轴——它无法区分高周期牛熊，会复现 2022 盲点。
- 组合为状态网格：{趋势↑·牛 / 趋势↑·熊(反弹陷阱) / 趋势↓·牛 / 趋势↓·熊 / 震荡·牛 / 震荡·熊}。
- 所有分类仅用 ≤t 数据（无前视）。
- 实现：`pandas-ta.adx()` + 纯 numpy ER（~10行）+ 高周期方向（纯计算），均无需重型依赖。

### 10.4 Purged / Embargo 交叉验证（写入硬纪律）

参数优化时**禁止使用朴素 KFold**。时间序列 CV 必须：

- 训练集与测试集之间 **purge** 掉标签重叠的样本（三屏障标签跨多根K线，天然重叠）；
- 测试集之后加 **embargo** 缓冲期，防止信息从测试期泄漏回训练。
- 现有 Walk-Forward（按时间切割）方向正确，但持仓/标签重叠较多时必须加 purge+embargo，否则高估样本内 Sharpe。
- 实现优先 `timeseriescv` / `skfolio` 的 PurgedKFold 或自实现；若后续升级 VectorBT PRO 则用其内置功能（属付费项，需 Founder 确认）。

### 10.5 报告新增必填字段（补充第八节）

- **三屏障触发分布**：止盈 / 止损 / 时间 各占比（**仅方向性事件策略**适用）。
- **状态门控描述指标**：覆盖率（通过门控信号占比）/ 保留率 / 状态间收益差异（**取代** 精确率/召回率，除非已训练独立二级模型）。
- **两轴状态归因**（涉及状态时）：各状态格的笔数/净P&L/胜率/平均R/MaxDD。

### 10.6 借鉴前置纪律（DEC-056，预登记前强制）

每个新信号/新策略设计前，预登记文档须先回答："**是否有成熟方法论或开源实现可借鉴？**"并记录借鉴对象与取舍。禁止在未检索成熟方案的情况下直接手写。
