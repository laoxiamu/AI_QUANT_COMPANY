## 3. 唯一变量精确定义：波动率缩仓系数

**恐惧度代理（市场级，单一，无前视）：**
- 取 BTC/USDT 4H 对数收益率，计算滚动 **180 根 K 线（=30 天）已实现波动率** σ_t（样本标准差）。
- 在信号 K 线 t_sweep 处，只使用 t_sweep 及之前的数据。
- 计算 σ_t 在**扩张窗口历史**（自数据起点至 t_sweep−1 的全部 σ 值）中的**百分位 p_t**。
- 预热：历史 σ 样本不足 250 个时，p_t 不可用 → 系数取 1.0（不缩仓）。
- 同一 BTC 恐惧度 gauge 应用于全部三品种（市场级恐惧，非单品种）。

**缩仓系数（二值，常数冻结）：**
```
若 p_t ≥ 0.66（高恐惧，波动率位于历史前 1/3）  →  vol_scalar = 0.5
否则                                          →  vol_scalar = 1.0
```

**最终仓位（采用 0B17 代码口径，DEC-046 修正 5）：**
```
risk_budget   = 0.01 × portfolio                  # 货币单位
risk_distance = ATR(14)[t_sweep] × 3.5            # 价格单位（不是百分比）
quantity      = risk_budget / risk_distance       # 数量
notional      = quantity × entry_price × vol_scalar
约束：单笔名义 ≤ 100% × portfolio；组合总敞口 ≤ 1x（超出按比例缩放）
```
> ⚠️ 注意单位：ATR(14) 为价格单位，`risk_budget / risk_distance` 得到的是【数量】，
> 必须 ×entry_price 才是名义。vol_scalar 作用于 quantity/notional（等价）。
> 禁止写成 `0.01×portfolio/risk_distance` 直接当名义（该式在价格单位下是数量）。

冻结常数：滚动窗口 180、百分位阈值 0.66、缩仓乘子 0.5、预热 250。**禁止据结果调整这四个常数**（调整即构成新假设，需重新预登记）。
